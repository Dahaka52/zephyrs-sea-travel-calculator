# План рефакторинга после аудита

Дата: 2026-03-13  
Проект: `zephyrs-sea-travel-calculator`

## 1. Краткий итог аудита

1. Единый расчёт и консистентность UI/API/чата обеспечены через `calculateTravelFromData`.
2. `lastInput` и `windowSettings` сохраняются в `client` scope, сохранение формы и окна дебаунсится.
3. UI дополняет блок «Вёсла» только для судов с `sailing.oars.available`.
4. Документация и smoke-чеклист актуализированы под текущее поведение.

## 2. Оставшиеся задачи (только актуальные)

P1:

1. Долокализовать макрос погоды: UI и чат-вывод, включая климатические зоны и описания.
2. Уточнить локализацию данных кораблей/условий при добавлении новых сущностей (поддерживать `*_en` как минимум).

P2:

1. Рассмотреть переход на стандартный `game.i18n` и `lang/*.json` для масштабирования локализаций.
2. Подготовить шаблон перевода для новых языков (структура ключей и процесс добавления).

## 3. Контроль качества (перед релизом)

1. `node tests/run-tests.js`
2. Smoke-чеклист `SMOKE_TEST_CHECKLIST_RU.md`.

## 4. Definition of Done для оставшихся задач

1. Макрос погоды локализован под выбранный язык.
2. Новые языки добавляются без изменения кода (через словари/ключи).
